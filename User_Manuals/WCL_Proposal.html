01 · Context

The Problem

A set of large Excel workbooks have grown organically across multiple data sources — manual entry, PDF extracts, and Google Sheets exports. They now contain a mix of format inconsistencies, duplicate IDs, missing required fields, and values that violate business rules. Before any downstream analysis or reporting can happen, the data needs a thorough, repeatable cleaning pass driven by explicit, configurable validation rules.

📋

Inconsistent Formats

Dates in 6 different layouts. Currency mixing EU and US separators. Text casing mixed across columns.

🔁

Duplicate IDs

Same record appearing under slightly different names or IDs, inflating counts and distorting analysis.

⚠️

Missing & Invalid Values

Required fields left blank, out-of-range numbers, invalid email addresses, and free-text in date columns.

📑

No Audit Trail

Changes are made ad-hoc with no record of what was fixed, when, or why — making reviews impossible.

🔗

Multi-Source Data

Records flow in from Excel files, PDF tables, and Google Sheets exports — each with slightly different structures.

📥

Manual Data Entry Risk

New rows are entered without validation, propagating the same formatting problems into future records.

02 · Solution

What WCL Delivers

WCL is a single .xlsm workbook containing a VBA macro engine driven entirely by a user-editable Field_Rules table. No code changes are needed to handle a new dataset — configure the rules table, point it at the source files, and click Run.

End result: You open one master workbook. Columns are consistently formatted, no duplicate IDs remain, all validation rules pass, and the Changelog tab documents every fix with a timestamp, before-value, after-value, and reason.

Core Output Guarantee

  • ✓
    Consistent column formats Every date column normalized to DD-MMM-YYYY. Currency to numeric with dot decimal. Text casing applied per rule.
  • ✓
    Zero duplicate IDs Exact-match deduplication on user-defined key columns, with fuzzy-name clustering for near-duplicates that differ only in spelling or legal suffix.
  • ✓
    All validation rules passing Required fields, type checks, range checks, and allowed-value lists all verified. Failures written to a Validation_Flags sheet for manual resolution.
  • ✓
    Changelog tab Every automated fix logged with: run date, change type, sheet, row, column, old value, new value, and reason code.
03 · Features

Full Feature Set

Data Import

  • 📂
    Multi-format file ingestion Open .xlsx, .xls, and .csv source files. Paste zone on the Staging sheet for PDF-extracted tables and Google Sheets exports.
  • 🔍
    Auto-detect header row Scans rows 1–5 to find the first row containing the required column names, handling reports with title rows or blank leading rows.
  • 🔤
    Column synonym matching Maps source columns to master columns by normalized name (handles "E-mail" / "Email" / "EMAIL" as the same column; dashes, spaces, and case ignored).
  • 🔢
    Scientific notation expansion Excel silently converts long IDs (e.g., vendor numbers) to scientific notation. WCL detects and expands these back to full numeric strings before any processing.

Column Formatting

  • 📅
    Flexible date parsing Tries multiple date formats (ISO, DMY, MDY, DD-MMM-YY, etc.) using locale-independent parsing. Writes a consistent DD-MMM-YYYY output. Flags unparseable dates for manual review.
  • 💰
    EU / US currency normalisation Detects whether commas and dots are decimal or thousands separators based on digit-group position. Strips currency symbols (€, $, £, ¥). Outputs clean numeric values.
  • 🔠
    Text standardisation Per-column rules for UPPER, lower, or ProperCase. Strips Unicode control characters, zero-width spaces, non-breaking spaces, and 8 exotic space variants.
  • 📧
    Email address validation & normalisation Validates structural format (@ + dot). Normalises semicolon/comma-delimited lists. Flags invalid addresses in Validation_Flags without removing them.
  • 🔢
    Formula-to-value conversion Replaces formula cells with their display text values before exporting to master — preserving leading zeros in IDs and formatted number strings.

Validation Engine

  • ✅
    Required field check Any column marked Required=YES in Field_Rules triggers a flag when the cell is blank or contains only whitespace.
  • 📏
    Type validation Checks that values in DATE, NUMBER, EMAIL, and TEXT columns actually conform to their declared type. Type mismatches are flagged with the expected type and the actual value found.
  • 📊
    Range & outlier checks Min / Max thresholds per column. Values outside the range are flagged as outliers for human review rather than silently modified.
  • 📋
    Allowed-values lists A comma-separated allowed-values list can be supplied in Field_Rules. Any value not in the list is flagged. Case-insensitive matching.
  • 🏷️
    Validation_Flags sheet Every failed check writes a row: Column, Row Reference, Rule Type, Expected, Actual. The reviewer corrects values here, then re-runs validation to confirm all pass.

Deduplication Engine

  • 🔑
    Exact-key deduplication User specifies one or more key columns in the Run sheet. Records sharing an identical composite key are collapsed, keeping the preferred row (configurable: first, latest date, or highest ID).
  • 🔀
    Fuzzy name clustering Groups name variants that differ by legal suffix (Ltd / Limited / LLC), word order ("US Bank" vs "Bank US"), or minor spelling. Uses a sliding-window Union-Find algorithm with complete-linkage guard to avoid false clusters.
  • 🧹
    Cluster-level deduplication Once clusters are built, the canonical representative (most frequent or first) is kept; all other variants are written to a Duplicates_Review sheet with their cluster ID for verification before deletion.
  • 📊
    Occurrence tracking Each retained record carries an OCCURRENCE_COUNT column showing how many source rows were merged into it — useful for spotting data-volume anomalies.

Guided Data Entry

  • 📝
    Validated entry form (frmWCLDataEntry) A UserForm bound to the Field_Rules config. Each field shows the correct format, dropdown lists for allowed-values columns, and red highlighting for invalid input before the row can be saved.
  • 🔒
    Convention enforcement at point of entry Field naming conventions, date formats, and casing rules are applied as the user types — not after. New records are always clean from day one.
  • 🔍
    Duplicate pre-check Before committing a new row, WCL checks the proposed key value against existing records and warns if a match or near-match already exists.

Reporting & Summary

  • 📊
    Auto-generated Summary tab Row counts, duplicate counts removed, validation pass/fail rates, date range of records, and a top-10 largest cluster table — generated after every clean run.
  • 🔽
    Filter & sort helpers One-click sort by any column. Include / exclude filter against a user-specified column value. Outputs a filtered view to a separate sheet without modifying master data.
  • 📤
    Clean archive export Exports Master + Summary + Changelog to a macro-free .xlsx named DD_MMM_YYYY_Clean.xlsx — safe to share without exposing VBA code.

Changelog & Audit Trail

  • 📒
    Structured Changelog tab Every automated change is logged: Run Date · Change Type · Sheet · Row · Column · Old Value · New Value · Reason Code. Appended on each run — never overwritten.
  • 🏷️
    Change types tracked FORMAT_DATE · FORMAT_CURRENCY · FORMAT_TEXT · DEDUP_REMOVED · DEDUP_MERGED · VALIDATION_FLAGGED · MANUAL_ENTRY · IMPORT
04 · Workflow

Full Clean Pipeline

A single click on Run Full Clean in the Run sheet executes the pipeline below in sequence. Each step can also be run individually. AppLock (screen updating off, calculation manual) is applied for the entire run by the Orchestrator.

1
Import

Open source file(s). Auto-detect headers. Paste to Staging sheet. Expand scientific notation.

2
Format

Normalize dates, currency, text casing per Field_Rules. Convert formulas to values.

3
Validate

Check required fields, types, ranges, allowed values. Write failures to Validation_Flags + Changelog.

4
Deduplicate

Exact-key dedup then fuzzy-name clustering. Move duplicates to review sheet. Log each removal.

5
Write Master

Write clean, deduplicated rows to Master sheet with consistent column order and formatting.

6
Summarise

Build Summary tab: counts, rates, top clusters, date ranges. Export clean .xlsx archive.


User review checkpoint: After step 3, the user inspects Validation_Flags and corrects any issues. After step 4, the user reviews Duplicates_Review before confirming deletions. WCL never silently discard data without a review trail.

PDF & Google Sheets Flow

  • 📄
    PDF source User copies the table from any PDF viewer and pastes into the Staging sheet paste zone. WCL picks it up from there identically to an Excel source. No PDF parsing library required.
  • 🌐
    Google Sheets source User exports the Sheet as .csv (File → Download → CSV). WCL imports CSV natively, opening all columns as text to preserve leading zeros. A companion Google Apps Script snippet can automate the export step.
05 · Architecture

Module Breakdown

Follows the same numbered-module pattern used across all projects in this repo. Each module has a single responsibility and all entry points flow through the Orchestrator.

Module Full Name Responsibility Layer
WCL0_Config Config Public Const + Public Type UDTs only. Sheet names, column names, change-type codes, validation UDT. No Sub, no Function, no business logic. Config
WCL1_Helpers Helpers Shared utilities: null-safe conversions, path joining, header lookups, column synonym resolver, safe sheet access. Thin wrappers over SharedLibrary modules. Utilities
WCL2_Importer Importer Open source files (xlsx/xls/csv). Auto-detect header row (scans rows 1–5). Map source columns to master columns via synonym lookup. Paste into Staging sheet. Expand scientific notation on ID columns. Step
WCL3_Formatter Formatter Column-by-column format pass driven by Field_Rules table. Calls date parser, currency normalizer, text casing engine. Converts formulas to values. Logs each change to Changelog. Step
WCL4_Validator Validator Reads Field_Rules. Checks required fields, type conformance, min/max range, allowed-values list, email format. Writes failures to Validation_Flags sheet and Changelog. Returns pass/fail counts. Step
WCL5_Deduplicator Deduplicator Phase 1: exact-key dedup using Dictionary keyed on composite ID column(s). Phase 2: fuzzy-name clustering via sorted sliding-window Union-Find with complete-linkage guard. Moves duplicates to review sheet. Logs removals. Step
WCL6_DataEntry DataEntry Launches frmWCLDataEntry. Builds form fields dynamically from Field_Rules (dropdowns for allowed-values columns, date pickers for date columns). Validates on Save. Pre-checks for duplicate key before commit. Step
WCL7_Reporter Reporter Generates Summary tab (counts, rates, cluster stats, date ranges). Sort-by-column helper. Include/exclude filter helper (writes filtered view to separate tab). Builds archive .xlsx export. Step

WCL9_Orchestrator Orchestrator All public button entry points. Only module that calls AppLock. Entry points: RunFullClean · RunValidateOnly · RunDedupeOnly · RunImport · AddNewRecord · GenerateSummary · ExportArchive · SetupWorkbook. Orchestrator

06 · Workbook Layout

Sheet Structure

Eight sheets. Three are user-facing, three are engine-internal, two are output.

Run

Buttons + config cells: source folder path, ID column(s), archive path, option flags. Never contains data.

Field_Rules

User-editable config table: Column Name · Data Type · Required · Min · Max · Allowed Values · Casing Rule. Drives every engine step.

Staging

Raw import paste zone. Source files land here. Never modified by the user directly — cleared and reloaded on each import.

Master

Clean, validated, deduplicated output. Single source of truth. Consistently formatted columns. AutoFilter enabled.

Validation_Flags

Rows failing validation: Column · Row Ref · Rule · Expected · Actual. User corrects values here, then re-runs validate to confirm.

Duplicates_Review

Duplicate records before deletion. Cluster ID + reason shown. User confirms before WCL removes from Master.

Summary

Auto-generated after each run: row counts, duplicate counts, validation pass rate, date range, top clusters table.

Changelog

Append-only audit log. Run Date · Change Type · Sheet · Row · Column · Old Value · New Value · Reason. Never cleared automatically.

07 · Reusable Assets

Library Gems Being Reused

Rather than writing utilities from scratch, WCL will import the following battle-tested modules from the SharedLibrary and VendorNormalizationEngine projects already in this repo. These are the specific functions that directly solve WCL's hardest problems.

SharedLibrary · LIB1_StringUtils

Unicode-Aware Text Cleaning

  • GF_StrictClean — strips 8 exotic Unicode space variants, zero-width chars, control chars. Essential for copy-paste from PDFs.
  • NormalizeForFuzzy — strips honorifics, lower-cases, replaces punctuation. Pre-step for dedup scoring.
  • NormalizeHeaderText — normalizes Unicode dashes to hyphens for column-agnostic matching.
SharedLibrary · LIB2_SheetUtils

Smart Header & Sheet Utilities

  • DetectHeaderRow — scans rows 1–5 for required column names. Handles reports with title rows above data.
  • MapHeaderColumns — returns a case-insensitive Dictionary of header → column index for synonym matching.
  • ConvertFormulasToValues — replaces formulas with display text, preserving leading zeros in IDs.
  • GetColByHeader — tries a list of synonyms to find a column (e.g., "Email" / "E-mail" / "EMAIL").
SharedLibrary · LIB3_DateUtils

Locale-Independent Date Parsing

  • TryParseDate — attempts to parse any date string; returns False on failure without raising an error.
  • FormatDateLong — always outputs DD-MMM-YYYY regardless of Windows locale settings.
  • ValidateRunDate — checks date is >= today for future-dated field validation.
SharedLibrary · LIB7_NumericUtils

Currency & Number Normalisation

  • GF_NormalizePriceToUS — detects EU vs US separator conventions by digit-group position. Handles multi-dot and multi-comma formats.
  • CanonicalizeAccountStr — strips non-digits, expands scientific notation, removes ".00" decimals. Critical for ID column dedup.
  • ExpandScientificNotation — recovers long vendor numbers Excel converted to 1.23E+11.
SharedLibrary · LIB4_FuzzyMatch · VNE2_Helpers

Enhanced Fuzzy Scoring Engine

  • ScorePair (VNE2 enhanced) — strips common suffix tokens before scoring to prevent false clusters. Adds prefix-only boost ("US Bank" vs "US Bank NA"). Does NOT boost mid-string containment.
  • CleanCompanyName — removes LLC, Ltd, Inc, Corp, The as whole-word tokens for fair comparison.
  • TokenSortKey — alphabetically reorders tokens to catch "US Bank" vs "Bank US" as a match.
VendorNormalizationEngine · VNE5_NameCluster

Union-Find Clustering Algorithm

  • Sorted sliding-window (default window=150) — O(n × window) instead of O(n²) all-pairs.
  • Two-pass: sort by cleaned name, then by token-sort key — catches both direct similarity and word-order variants.
  • Complete-linkage guard — verifies cluster roots score above threshold before merging, preventing chain-explosion false clusters.
  • Path-compressed Union-Find — O(α(n)) amortized per merge operation.
VendorNormalizationEngine · VNE3_Setup

Flexible Source File Opening

  • Blank filename → reads from ThisWorkbook. Named file → opens externally ReadOnly.
  • Auto-detect sheet by required headers — no dependency on a fixed sheet name.
  • CSV forced-text mode — opens all columns as text to preserve leading zeros.
SharedLibrary · LIB5_Logging

Structured 9-Column Audit Log

  • GF_LogRowAction — logs macro name, source file, sheet, row, index value, decision, outcome, and actions taken. Exactly what WCL's Changelog needs.
  • Auto-creates Log sheet on first write. Never crashes if sheet is missing.
  • SafeLog — lightweight 3-column event tracing for step-level diagnostics.
SharedLibrary · LIB8_HtmlUtils

HTML Email & Export Generation

  • IsValidEmail — structural email validation for the email-column checker.
  • PlainToHtmlRich — convert plain-text notes (bullet lists, paragraphs) to formatted HTML for summary emails.
  • WrapHtml — full HTML document wrapper with UTF-8 meta for exported reports.
VendorNormalizationEngine · VNE9_Archive

Clean Export with Summary Sheet

  • Pattern for copying selected sheets to a new macro-free .xlsx.
  • Auto-versioning if file already exists (_V1, _V2, etc.).
  • Summary sheet builder: cluster stats, match breakdown, top-10 table — adapted for WCL's clean-run stats.
VendorNormalizationEngine · VNE4_OracleReduce

Array-Based Deduplication at Scale

  • Load entire dataset into memory array — no cell-by-cell loops.
  • Dictionary keyed on Chr(0)-separated composite key — handles 500k rows in seconds.
  • Occurrence counting — tracks how many source rows merged into each retained record.
  • Progress callbacks every 25k rows — keeps UI responsive on large files.
