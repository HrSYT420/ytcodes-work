Python Automation

Excel handles a lot. It does not handle everything. Web scraping, API calls, large-file ETL, and long-running batch jobs sit outside the spreadsheet's comfort zone.

I build Python scripts and small services for that work. Output lands back in the format your team actually uses, which is often an Excel file. The automation runs on a schedule or on demand, headless, on the machine that makes sense for the workload.
